# DentalStock 작업 보고서

작성일: 2026-05-20

## 버전 관리

- 현재 작업 전 상태는 `codex-current-snapshot` 브랜치에 `chore: snapshot current dentalstock work` 커밋으로 보존했습니다.
- 수정 작업은 `codex-fix-user-test-issues` 브랜치에서 진행했습니다.
- `.claude/`, `COMMIT_MESSAGE.txt`, `COMMIT_MSG.txt`, `COMMIT_MSG_VENDOR.txt`는 임시 파일로 판단해 커밋 대상에서 제외했습니다.

## 주요 수정

### 발주/배송 플로우

- pending 발주에서 바로 `추적 시작`으로 ordered 상태가 되는 우회 경로를 제거했습니다.
- 관리자/원장에게만 `승인`, `반려`, `송장 등록`, `입고 확인` 액션을 노출하도록 정리했습니다.
- 위생사는 자기 발주 요청을 볼 수 있지만 승인/송장 등록/입고 확인은 할 수 없도록 UI와 훅 레벨에서 모두 막았습니다.
- 승인 후에는 진행중 탭에서 `송장 등록`을 할 수 있고, 송장 등록 후 `송장 상세`를 볼 수 있게 했습니다.
- 반려된 발주를 확인할 수 있도록 배송추적에 `반려` 탭을 추가했습니다.

### 오버레이/모달

- 품목 상세 화면에서 발주 요청 시트를 열면 뒤쪽 버튼이 클릭을 가로채던 z-index 문제를 수정했습니다.
- `BottomSheet`가 상세 오버레이보다 위에 오도록 정리해 실제 `발주 요청 보내기` 버튼이 정상 클릭됩니다.

### 날짜 처리

- `todayKey()`가 UTC 날짜를 쓰던 문제를 고쳐 `Asia/Seoul` 기준 날짜를 사용하도록 변경했습니다.
- 홈의 오늘 수술 카드와 입출고의 오늘/어제 표시가 한국 날짜 기준으로 동작합니다.

### 홈/문구 정리

- 홈의 `주문 승인 대기`, `배송 완료`, `주문 링크`처럼 실제 상태와 어긋나던 문구를 정리했습니다.
- pending은 `발주 승인 대기`, ordered는 `입고 대기`로 표시합니다.
- 가격이 없는 항목은 `0원` 대신 `가격 미등록`으로 표시합니다.

### 입출고 UX

- 입출고 품목 선택 시트에 `품목명 검색` 입력을 추가했습니다.
- 현재 재고보다 많은 출고 수량을 입력하면 `재고 부족` 상태로 제출 버튼을 비활성화합니다.
- 훅에서도 과출고를 실제로 막아 재고가 임의로 0 처리되는 일을 없앴습니다.

### 관리 화면

- `품목 추가`, `초기 재고 일괄 입력`, `초기 데이터 리셋`을 직원 관리에서 빼고 `품목 관리` 탭으로 분리했습니다.
- 도매 설정에서 사이트별 ID/비밀번호 입력 필드를 추가했습니다.
- 도매 설정 안내 문구를 현재 로컬 저장 기반의 테스트용 설정임을 알 수 있게 조정했습니다.

### 배송 상세

- 가격 데이터가 없는 주문에서 `원`만 표시되거나 `undefined원`처럼 보이는 문제를 제거했습니다.
- 배송 타임라인 더미 상태를 ordered/received 상태에 맞춰 자연스럽게 정리했습니다.
- 화면 텍스트에 섞여 있던 불필요한 기호성 표시를 줄였습니다.

## 실제 브라우저 검증 항목

- 원장 로그인 후 오늘 수술 카드가 한국 날짜 기준으로 노출되는지 확인
- 홈에서 pending/ordered 주문 문구가 실제 상태와 맞는지 확인
- pending 발주에서 `추적 시작` 우회가 사라졌는지 확인
- 관리자 승인 후 송장 등록과 송장 상세가 정상 동작하는지 확인
- 배송 상세에서 결측 가격과 타임라인 표시가 깨지지 않는지 확인
- 위생사 계정에서 승인/송장 등록 권한이 노출되지 않는지 확인
- 품목 상세에서 발주 요청 시트가 실제로 클릭 가능한지 확인
- 입출고 품목 검색과 과출고 차단이 동작하는지 확인
- 품목 관리 탭과 도매몰 계정 입력 필드가 노출되는지 확인

## 추가 개선

사용자 요청에 따라 버그 수정 외에도 한 번 더 제품 관점의 개선을 진행했습니다.

- 하단 내비게이션과 헤더의 `배송추적` 명칭을 실제 기능 범위에 맞춰 `발주/배송`으로 변경했습니다.
- 발주/배송 탭 배지를 pending만이 아니라 pending/ordered 활성 업무 전체 기준으로 계산하도록 개선했습니다.
- 홈의 `오늘 해야 할 일` 카드는 그룹별 반복 문구 대신 실제 개별 업무 목록처럼 보이도록 재구성했습니다.
- 홈 업무 카드의 건수는 그룹 개수가 아니라 실제 처리할 작업 수를 표시합니다.
- 출고 화면의 증가 버튼과 빠른 수량 버튼 색상을 입고/출고 맥락에 맞춰 파란색/빨간색으로 구분했습니다.
- 관련 테스트를 새 UX 기준으로 갱신했고, 동일한 125개 테스트 전체 통과를 재확인했습니다.

## 마지막 코드 다이어트

자기 전 맡긴 최종 정리 요청에 맞춰 기능은 유지하고 코드 반복과 불필요한 연결을 줄였습니다.

- `ShippingOrderCard`의 반복되던 카드 본문, 상태 헤더, 액션 버튼 스타일을 작은 공통 컴포넌트/상수로 정리했습니다.
- 발주/배송 화면의 권한별 주문 표시 기준을 `canApprove` 하나로 통일해 역할 조건 중복을 제거했습니다.
- 입출고 시트의 입고/출고 색상 분기를 `accent` 값으로 모아 같은 삼항식이 여러 번 반복되지 않게 했습니다.
- 실제로 쓰지 않는 `AdminScreen`, `OrderRequestSheet`, `ReceiptConfirmSheet` props 전달을 제거했습니다.
- 최종 정리 후 `npm run lint`, `npm test`, `npm run build`, Playwright 21개 실제 흐름 검증을 모두 다시 통과했습니다.

## Toss 디자인 시스템 적용

제공된 Toss/TDS 문서 기준으로 현재 앱에 적용 가능한 디자인 디테일을 새 브랜치에서 반영했습니다.

- Toss 토큰 값을 문서 기준에 맞춰 조정했습니다: `blue600`, `blue50`, warm grey scale, green/orange/yellow/teal/purple semantic 색상.
- 카드, 배지, 입력, segmented tab, toast, bottom navigation의 radius, shadow, padding, font weight를 TDS Mobile 규칙에 맞게 다듬었습니다.
- 재고 수량, 입출고 수량, 통계 값, 분석 숫자 등 데이터성 숫자에 monospace + tabular numeral 표현을 적용했습니다.
- 카테고리/탭 선택처럼 사용자가 누르는 인터랙션은 Toss Blue 중심으로 정리하고, red/orange/green은 상태와 위험 의미에 집중하도록 조정했습니다.
- 입출고, 재고 상세, 알림, 배송, 홈 업무 카드, 관리자 분석 화면의 버튼과 요약 카드 밀도를 모바일 375px 기준에 맞게 재정렬했습니다.
- 적용 후 `npm run lint`, `npm test`, `npm run build`, Playwright 21개 실제 흐름 검증을 모두 통과했습니다.

## 후속 UX 보완

사용자가 추가로 짚은 실제 사용 불편 사항을 반영했습니다.

- 입출고 등록에서 품목 선택 리스트 상단의 닫기 버튼을 36px 원형 X 버튼으로 고정해, 리스트 위에서 바로 닫을 수 있게 했습니다.
- 발주/배송 상단 필터 버튼은 모바일 폭에서 글자가 위아래로 갈라지지 않도록 아이콘과 버튼 내부 배지를 덜어내고, 선택된 탭 아래에 `승인대기 1건`처럼 현재 건수만 표시하도록 조정했습니다.
- 발주/배송 카드 액션 버튼도 `송장 등록`, `입고 확인`, `송장 상세` 같은 라벨이 한 줄로 유지되도록 버튼 폭/줄바꿈/아이콘 축소 규칙을 정리했습니다.
- 관리 > 수술 준비에서 수술 일정 삭제 기능을 추가했습니다. 삭제 시 확인창을 띄우고, 관련 수술 알림과 리마인더 상태도 함께 정리합니다.
- 예정 수술이 없을 때 빈 상태 문구를 추가했습니다.
- 표적 Playwright 검증으로 닫기 버튼 노출, 발주/배송 버튼 한 줄 유지, 수술 일정 등록/삭제를 확인했고, 전체 21개 브라우저 흐름도 다시 통과했습니다.
