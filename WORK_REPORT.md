# DentalStock 작업 보고서

작성일: 2026-05-20

## 버전 관리

- 현재 작업 전 상태는 `codex-current-snapshot` 브랜치에 `chore: snapshot current dentalstock work` 커밋으로 보존했습니다.
- 수정 작업은 `codex-fix-user-test-issues` 브랜치에서 진행했습니다.
- `.claude/`, `COMMIT_MESSAGE.txt`, `COMMIT_MSG.txt`, `COMMIT_MSG_VENDOR.txt`는 임시 파일로 판단해 커밋 대상에서 제외했습니다.

## 주요 수정

### 발주/배송 플로우

- pending 발주에서 바로 `추적 시작`으로 ordered 상태가 되는 우회 경로를 제거했습니다.
- 관리자/원장에게만 `승인`, `반려`, `송장 등록`, `입고 확인` 액션을 노출하도록 정리했습니다.
- 위생사는 자기 발주 요청을 볼 수 있지만 승인/송장 등록/입고 확인은 할 수 없도록 UI와 훅 레벨에서 모두 막았습니다.
- 승인 후에는 진행중 탭에서 `송장 등록`을 할 수 있고, 송장 등록 후 `송장 상세`를 볼 수 있게 했습니다.
- 반려된 발주를 확인할 수 있도록 배송추적에 `반려` 탭을 추가했습니다.

### 오버레이/모달

- 품목 상세 화면에서 발주 요청 시트를 열면 뒤쪽 버튼이 클릭을 가로채던 z-index 문제를 수정했습니다.
- `BottomSheet`가 상세 오버레이보다 위에 오도록 정리해 실제 `발주 요청 보내기` 버튼이 정상 클릭됩니다.

### 날짜 처리

- `todayKey()`가 UTC 날짜를 쓰던 문제를 고쳐 `Asia/Seoul` 기준 날짜를 사용하도록 변경했습니다.
- 홈의 오늘 수술 카드와 입출고의 오늘/어제 표시가 한국 날짜 기준으로 동작합니다.

### 홈/문구 정리

- 홈의 `주문 승인 대기`, `배송 완료`, `주문 링크`처럼 실제 상태와 어긋나던 문구를 정리했습니다.
- pending은 `발주 승인 대기`, ordered는 `입고 대기`로 표시합니다.
- 가격이 없는 항목은 `0원` 대신 `가격 미등록`으로 표시합니다.

### 입출고 UX

- 입출고 품목 선택 시트에 `품목명 검색` 입력을 추가했습니다.
- 현재 재고보다 많은 출고 수량을 입력하면 `재고 부족` 상태로 제출 버튼을 비활성화합니다.
- 훅에서도 과출고를 실제로 막아 재고가 임의로 0 처리되는 일을 없앴습니다.

### 관리 화면

- `품목 추가`, `초기 재고 일괄 입력`, `초기 데이터 리셋`을 직원 관리에서 빼고 `품목 관리` 탭으로 분리했습니다.
- 도매 설정에서 사이트별 ID/비밀번호 입력 필드를 추가했습니다.
- 도매 설정 안내 문구를 현재 로컬 저장 기반의 테스트용 설정임을 알 수 있게 조정했습니다.

### 배송 상세

- 가격 데이터가 없는 주문에서 `원`만 표시되거나 `undefined원`처럼 보이는 문제를 제거했습니다.
- 배송 타임라인 더미 상태를 ordered/received 상태에 맞춰 자연스럽게 정리했습니다.
- 화면 텍스트에 섞여 있던 불필요한 기호성 표시를 줄였습니다.

## 실제 브라우저 검증 항목

- 원장 로그인 후 오늘 수술 카드가 한국 날짜 기준으로 노출되는지 확인
- 홈에서 pending/ordered 주문 문구가 실제 상태와 맞는지 확인
- pending 발주에서 `추적 시작` 우회가 사라졌는지 확인
- 관리자 승인 후 송장 등록과 송장 상세가 정상 동작하는지 확인
- 배송 상세에서 결측 가격과 타임라인 표시가 깨지지 않는지 확인
- 위생사 계정에서 승인/송장 등록 권한이 노출되지 않는지 확인
- 품목 상세에서 발주 요청 시트가 실제로 클릭 가능한지 확인
- 입출고 품목 검색과 과출고 차단이 동작하는지 확인
- 품목 관리 탭과 도매몰 계정 입력 필드가 노출되는지 확인
